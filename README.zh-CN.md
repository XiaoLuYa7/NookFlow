# NookFlow

[English](README.md) | [简体中文](README.zh-CN.md)

NookFlow 是一款轻量级 macOS 灵动岛应用，使用 SwiftUI 和 AppKit 构建。它会在菜单栏附近显示一个适配刘海区域的紧凑面板，并可展开为一个聚合音乐、歌词、提醒事项、日历、天气、文件、快捷应用和快捷指令的控制界面。

应用设计目标是贴近 macOS 原生体验：空闲时尽量降低资源占用，面板切换保持顺滑，窗口焦点行为可预期，并清晰处理各类系统权限边界。

## 功能

- macOS 紧凑与展开两种灵动岛面板形态。
- 音乐播放信息、播放控制、封面和同步歌词。
- 桌面歌词窗口与刘海区域歌词展示。
- 日历、提醒事项、天气、待办、文件暂存、快捷应用启动和快捷指令模块。
- 通用行为、通知、快捷应用和快捷指令设置页面。
- 基于 SwiftPM 的核心解析与展示逻辑回归测试。

## 环境要求

- macOS 14 或更高版本。
- 推荐使用 Xcode 16 或更高版本。
- Swift 6 工具链。

部分功能会根据启用的模块请求 macOS 权限，例如定位、日历、提醒事项、自动化或辅助功能权限。

## 构建与运行

打开 Xcode 项目：

```sh
open NookFlow.xcodeproj
```

然后选择 `NookFlow` scheme 并运行应用。

也可以使用 Swift Package Manager 运行回归测试：

```sh
swift test
```

## 项目结构

```text
App/              应用入口与应用代理
Panel/            AppKit 窗口和面板控制器
Views/            SwiftUI 灵动岛、设置、歌词和模块视图
Models/           状态、数据提供、解析、播放、文件和设置逻辑
Assets.xcassets/  应用图标和内置视觉资源
Tests/            SwiftPM 回归测试
TestsSupport/     测试支持类型
```

## 开发说明

- 灵动岛状态更新应尽量采用事件驱动和增量更新。
- 普通 UI 刷新避免高频轮询。
- 面板关闭或模块停用时，应取消对应任务、定时器和监听器。
- 文件扫描、网络请求和大量解析工作不要放在主线程执行。
- 缓存应有明确边界，并由对应功能模块管理生命周期。
